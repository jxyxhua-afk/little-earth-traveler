# Global Country Game Design Framework

## 1. 项目新定位

《小小地球旅行家》不是国家换皮小游戏合集，也不是儿童百科软件。

它是一款儿童通过地球仪进入不同国家，在可互动、可触摸、可碰撞、可探索的 3D 场景中认识世界差异的互动认知游戏。

地球仪是世界入口。国家关卡不是单一任务页，而是一个能让孩子认识这个国家的互动小世界。游戏目的不是为了闯关，而是让孩子通过玩，认识不同国家的建筑、动物、交通、食物、风景和文化符号。

核心体验是：

```text
玩中识物
玩中理解差异
玩中建立世界地图感
```

孩子玩完一个国家后，应该能记住这个国家的 3-5 个明显符号，而不是只记住自己完成了一个普通小游戏。

## 2. 原方案问题

之前的美国关卡暴露了一个根本问题：先设计小游戏，再套国家皮肤。

这种做法会让国家主题被玩法绑架。比如把美国简化成校车、马路和篮球，会导致孩子只是在玩一个驾驶投篮小游戏，却没有真正认识美国和中国、日本、埃及、澳洲有什么不同。

必须禁止以下方向：

- 不能先想一个小游戏，再把国家元素贴上去。
- 不能用一个校车或一个篮球代表美国。
- 不能让国家关卡变成没有辨识度的普通任务页。
- 不能为了游戏而游戏。
- 不能把文化物体只当装饰。
- 不能用文字解释替代孩子的视觉识别和操作体验。

正确方向是：

```text
先定义这个国家最适合孩子认识的世界特征
再把它转化成可看的场景、可碰的物体、可玩的动作
```

## 3. 新设计顺序

每个国家关卡必须按下面顺序设计：

```text
国家认知目标
↓
国家标志性场景
↓
主物体
↓
核心动作
↓
物理互动
↓
文化 / 地理 / 生活认知反馈
↓
图鉴收集
```

这意味着国家关卡不是从“我要做一个投篮游戏”开始，而是从“美国最适合 4-7 岁孩子记住什么”开始。

玩法必须服务于国家认知。动作、物体、声音、成功反馈都要共同回答一个问题：

```text
孩子通过这个国家关卡，多认识了这个国家的什么？
```

## 4. 每个国家关卡必须回答的问题

开发任何国家前，必须先回答这 6 个问题：

| 问题 | 设计要求 |
| --- | --- |
| 这个国家最想让孩子记住什么？ | 不是成年人知识，而是 4-7 岁孩子能识别、能复述的东西。 |
| 第一眼能看到哪些国家符号？ | 建筑、动物、交通、食物、自然、节日、生活方式。 |
| 主场景是什么？ | 不是平面背景，而是能装下国家符号的 3D 玩具空间。 |
| 孩子能操作什么？ | 主物体必须形象明确，比如校车、火车、雪橇、热气球、船、动物。 |
| 动作和国家主题有什么关系？ | 不能为了动作而动作，动作必须来自这个国家的文化或地理符号。 |
| 玩完后孩子多知道了什么？ | 必须有清晰的认知收获，并通过图标、点亮、声音、收集反馈出来。 |

如果这 6 个问题答不上来，这个国家不能进入开发。

## 5. 每个国家的最低标准

每个国家关卡最低必须包含：

- 至少 3 个国家标志性元素。
- 至少 1 个主物体。
- 至少 1 个核心动作。
- 至少 1 种物理反馈。
- 至少 1 个声音反馈。
- 至少 1 张国家发现卡。
- 孩子不读文字也能知道自己在玩哪个国家。

进一步要求：

- 国家符号不能只停留在装饰层。
- 每个符号至少对应一个可看、可碰、可触发、可点亮的反馈。
- 主物体必须从外形上可识别，不能只是抽象方块。
- UI 不能依赖长文字说明，核心操作要用图标、形状和物理隐喻表达。
- 成功反馈应该是“发现了某个国家符号”，不是单纯分数 +1。

## 6. 美国关卡新版方向

### 名称

```text
美国 · 校车小镇探索
```

### 核心认知

美国有黄色校车、公路文化、自由女神像、好莱坞电影文化、篮球场、汉堡 / 热狗等常见食物，也有摩天楼、大桥等城市景观。

美国关卡不应该叫“校车篮球街区”，因为这个名字太窄。篮球只是美国的一个发现点，不是整个关卡的全部。

### 国家符号

美国关卡应优先包含：

- 黄色校车
- 自由女神像
- 好莱坞牌
- 篮球场
- 汉堡 / 热狗摊
- 66 号公路
- 摩天楼 / 金门大桥远景可选

### 主场景

一个 3D 美国小镇玩具地图：

```text
左侧：校车站
中间：66 号公路小路
上方：自由女神像小广场
右上：好莱坞牌山坡
右下：篮球场
下方：汉堡 / 热狗摊
远景：摩天楼 + 星条旗 + 大桥轮廓
```

### 核心流程

```text
开校车
↓
沿 66 号公路探索
↓
发现自由女神像
↓
露出好莱坞牌
↓
完成汉堡摊小互动
↓
到篮球场投篮
↓
点亮美国发现卡
```

### 符号到玩法的转化

| 国家符号 | 可操作动作 | 认知反馈 |
| --- | --- | --- |
| 黄色校车 | 驾驶、停靠、转向、避障 | 美国小朋友常见黄色校车。 |
| 自由女神像 | 推火炬积木、点亮火炬 | 美国有自由女神像。 |
| 好莱坞牌 | 撞开纸箱，露出 HOLLYWOOD 字牌 | 美国有电影文化。 |
| 汉堡 / 热狗摊 | 推动或叠放食材积木 | 美国有常见快餐文化。 |
| 篮球场 | 拖拽篮球投篮 | 美国有篮球运动文化。 |
| 66 号公路 | 开校车穿过路牌和小障碍 | 美国有公路文化。 |

### 美国发现卡

美国关卡完成后，图鉴里应该点亮：

```text
□ 黄色校车
□ 自由女神像
□ 好莱坞牌
□ 汉堡 / 热狗摊
□ 篮球场
```

4 岁孩子不一定读字，所以发现卡必须以图标为主，文字为辅。

## 8. UI 新原则

旧 UI 容易变成任务计数：

```text
篮球 0/3
障碍 3/3
```

这会把游戏拉回普通网页小游戏。新 UI 应该围绕“发现国家”设计。

### UI 应改成国家发现卡

例如美国：

```text
美国发现
□ 校车
□ 自由女神像
□ 好莱坞
□ 汉堡摊
□ 篮球场
```

实际呈现时，应以图标为主，文字为辅：

- 校车图标
- 自由女神像图标
- 好莱坞牌图标
- 汉堡 / 热狗图标
- 篮球图标

### UI 规则

- 少文字，大图标。
- 当前目标要非常明显。
- 核心按钮必须像真实物体，比如方向盘、油门、图鉴卡，而不是纯文字按钮。
- 完成一个国家符号后，场景里要点亮、出现声音、收集到图鉴。
- UI 不应该盖住主要 3D 场景。
- 4 岁孩子即使不识字，也应该能通过图标和画面理解下一步。

## 9. 世界观设计

这款游戏应该帮助孩子形成的不是：

```text
我去美国打了个篮球
```

而是：

```text
世界上不同国家有不同的建筑、交通、食物、动物、风景和生活方式。
我可以通过玩，去发现这些不同。
```

世界观核心是：

```text
地球是一个可以探索的大玩具。
每个国家都有自己的颜色、声音、建筑、动物、食物和玩法。
孩子不是在闯关，而是在发现世界。
```

## 10. 验收标准

以后任何国家关卡都必须通过下面的检查：

### 国家辨识度

- 孩子第一眼能不能看出这是哪个国家？
- 是否至少有 3 个清晰的国家符号？
- 这些符号是否从外形上可识别，而不是靠文字解释？

### 认知收获

- 孩子玩完能不能说出 3 个这个国家的东西？
- 是否知道这个国家和其他国家有什么不同？
- 是否从场景、声音、物体和动作中获得认知，而不是从说明文字里获得？

### 玩法来源

- 玩法是否来自国家主题，而不是随便套进去？
- 主物体、动作、声音、物理反馈是否一致？
- 是否避免了“车 + 球 + 方块 + 斜坡”的重复模板？

### 儿童操作

- 4 岁孩子不读文字是否也能大概知道怎么玩？
- 核心操作是否使用图形、物体和物理隐喻？
- 控制是否足够直接、可触摸、可试错？

### 产品质感

- 是否避免了普通网页小游戏感？
- 是否有真实 3D 空间、体积、阴影和前后关系？
- UI 是否是辅助，而不是主视觉？

如果一个国家关卡不能让孩子记住这个国家，它就不能通过验收。

## 11. Codex 修改边界

后续修改必须遵守：

- 先写国家设计卡，再写代码。
- 先确定国家认知目标，再确定玩法。
- 不允许把失败的关卡代码和正确的设计文档混在同一个 checkpoint。
- 每次只做一个小闭环，验收通过后再进入下一步。
- 不允许为了快速实现而回到纯 HTML/CSS 平面小游戏。
- 不允许只做物理效果而忽略国家识别。
- 不允许扩展新国家前，旧国家还没有真实试玩通过。

建议开发顺序：

```text
总设计框架
↓
美国新设计卡
↓
美国 3D 小镇骨架
↓
美国符号发现卡
↓
美国各符号互动
↓
美国真实试玩
↓
再重做或扩展其他国家
```

这条原则优先级高于任何局部玩法实现。
